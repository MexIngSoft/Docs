# Estandar de nombres de proyecto

## Regla

Cada proyecto tiene un nombre comercial, un nombre tecnico y un
`ApplicationCode` canonicos. No se crean variantes nuevas en rutas, carpetas,
APIs, variables o textos visibles.

Para REFAPART:

```text
Nombre comercial: REFAPART
Nombre tecnico: RefaPart
ApplicationCode: REFAPART
Ruta API: /api/v1/projects/refapart/
```

`Refakto`, `Refapart`, `Refa-Part` y `RefaParts` son aliases historicos. No se
usan en desarrollo nuevo. Las referencias historicas se archivan; no gobiernan
contratos vigentes.
