# Checklist PR documental

- [ ] El documento tiene proposito.
- [ ] Esta en la carpeta canonica correcta.
- [ ] No duplica un documento existente.
- [ ] Enlaza documentos relacionados.
- [ ] Usa plantilla si es documento nuevo.
- [ ] Define owner, estado y prioridad cuando aplique.
- [ ] No mezcla Core con reglas de empresa.
- [ ] Actualiza `_meta` si cambia navegacion o relaciones.
- [ ] Conserva fuentes historicas en `_archive`.
- [ ] Usa `PENDIENTE_DE_VALIDACION` cuando falte confirmacion.
