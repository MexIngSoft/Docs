# Flujo frontend de cotizacion

1. Cliente selecciona solucion.
2. Contesta formulario.
3. Projects arma propuesta.
4. Rules Engine evalua reglas.
5. Pricing calcula precio.
6. Logistics estima envio.
7. Quote/Sales genera cotizacion.
8. Cliente descarga, solicita revision o aprueba.

## Salidas de contacto

Durante el MVP, el flujo debe permitir:

- Enviar solicitud por formulario.
- Abrir WhatsApp directo con mensaje precargado.
- Registrar origen del lead.

El chat en sitio queda como fase futura, no como requisito del MVP inicial.
