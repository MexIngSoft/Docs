# Flujo de cotizacion del cliente

1. Cliente entra a la web.
2. Selecciona una solucion.
3. Elige o responde componentes/necesidades.
4. Gateway envia contexto a Projects.
5. Projects propone solucion.
6. Catalog valida productos.
7. Supplier aporta disponibilidad de proveedor cuando aplique.
8. Inventory valida stock propio.
9. Rules Engine aplica reglas.
10. Pricing calcula precio final.
11. Logistics calcula envio.
12. Quote API genera cotizacion.
13. Cliente descarga o acepta la cotizacion.
14. Cotizacion puede convertirse en pedido.
